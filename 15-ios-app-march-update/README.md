# Проект 15: Верификация исправлений и регрессионное тестирование

**Цель проекта:** Подтвердить корректность внесённых исправлений в приложении IVI и отсутствие регрессии смежного функционала на платформе iOS.

**Задачи проекта:**
* проверить удаление контента из блока «Продолжить просмотр» на главной странице;
* убедиться в работоспособности бесшовной авторизации при переходе Web → App;
* исключить регрессию смежного функционала.

---

## Объект тестирования

* **Платформа:** iOS
* **Устройство:** iPhone 14 Pro Max
* **Версия ОС:** iOS 18.2.1
* **Приложение:** IVI
* **Тип тестирования:** Manual QA
* **Формат:** Bug Fix Verification / Regression Testing

---

## Проверяемые сценарии

### 1. Удаление контента из блока «Продолжить просмотр»

Проверка будет выполнена в детском профиле на главной странице приложения.

**Сценарий включает:**
* наличие контента в блоке ПП;
* вызов контекстного меню;
* удаление контента;
* проверку фактического удаления.

**Ожидаемый результат:** Контент полностью удаляется из блока «Продолжить просмотр».

---

### 2. Бесшовная авторизация (Web → App)

Проверка передачи авторизации из мобильной веб‑версии ivi.ru в установленное приложение.

**Сценарий включает:**
* авторизацию пользователя в веб‑версии;
* переход в приложение через кнопку «Открыть в приложении»;
* проверку автоматической авторизации без повторного ввода данных.

**Ожидаемый результат:** Пользователь автоматически авторизуется в приложении.

---

### 3. Проверка отсутствия регрессии смежного функционала

Комплексная проверка смежных функций приложения после внесения изменений.

**Сценарий включает:**
* проверку навигации между разделами приложения;
* тестирование воспроизведения видеоконтента;
* верификацию работы поиска и фильтров;
* контроль корректности отображения персональных рекомендаций;
* проверку стабильности работы приложения при смене профилей.

**Ожидаемый результат:** Отсутствие сбоев и ошибок в работе смежных функций, стабильность производительности.

---

## Результат

По итогам тестирования планируется подтвердить:

* стабильность работы функции удаления контента из блока «Продолжить просмотр»;
* корректность функционирования бесшовной авторизации Web → App;
* отсутствие регрессии в работе смежных функций приложения.

Проект позволит оценить:
* эффективность верификации багфиксов;
* качество регрессионного тестирования;
* надёжность кросс‑платформенных пользовательских сценариев;
* общую стабильность приложения после внесения изменений.
