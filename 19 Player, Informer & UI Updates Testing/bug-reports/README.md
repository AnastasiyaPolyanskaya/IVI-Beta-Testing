# Bug Reports - Player, Informer & UI Updates

В данном разделе представлены баг-репорты, выявленные в ходе тестирования обновлений мобильного приложения.

Отчёты отражают реальные дефекты, обнаруженные при проверке информера аудиосериалов, плеера и UI-изменений.

---

## Цель

Зафиксировать найденные дефекты, описать условия их воспроизведения и передать информацию для исправления.

---

## Область дефектов

### 1. Информер аудиосериалов

- информер не отображается после "чистой" установки приложения;
- некорректное поведение нового функционала.

---

### 2. Плеер

- отсутствует перемотка тапом по экрану;
- нарушено базовое взаимодействие с контентом;
- проблемы воспроизводятся в разных типах плееров:
  - трейлеры;
  - мини-драмы.

---

### 3. UI и поведение

- некорректное отображение новых элементов;
- несоответствие ожидаемому поведению интерфейса.

---

## Структура баг-репорта

Каждый баг-репорт содержит:

- ID  
- Название  
- Окружение  
- Предусловия  
- Шаги воспроизведения  
- Ожидаемый результат  
- Фактический результат  
- Серьёзность (Severity)  
- Приоритет (Priority)  
- Комментарий  

---

## Окружение

Тестирование проводилось на:

- iPhone 14 Pro Max  
- iOS 26.6.4  

---

## Серьёзность дефектов

- Critical — блокирует работу приложения  
- Major — влияет на ключевой функционал  
- Minor — незначительные дефекты  
- Trivial — визуальные проблемы  

---

## Вывод

В ходе тестирования выявлены дефекты, влияющие на ключевые пользовательские сценарии:

- отображение нового функционала;
- взаимодействие с плеером;
- пользовательский опыт.

Особое внимание требуется уделить работе плеера и отображению информера, так как данные сценарии напрямую влияют на восприятие продукта пользователем.
