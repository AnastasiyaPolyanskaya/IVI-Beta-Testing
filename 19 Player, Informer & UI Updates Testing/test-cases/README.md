# Test Cases - Player, Informer & UI Updates

В данном разделе представлены тест-кейсы для проверки обновлений мобильного приложения, связанных с информером аудиосериалов, плеером и изменениями пользовательского интерфейса.

Тест-кейсы покрывают ключевые пользовательские сценарии и направлены на проверку стабильности, корректности отображения и удобства взаимодействия.

---

## Цель тестирования

Проверить корректность работы новых и обновленных компонентов приложения, а также убедиться в отсутствии ошибок после релиза.

---

## Область покрытия

### 1. Информер аудиосериалов

- отображение информера на чистой установке;
- корректность поведения при переходах внутри приложения;
- отсутствие смещения ("не улетает");
- отсутствие крашей.

---

### 2. Плеер

Проверено:

- работа сикбара (полосы прокрутки);
- отображение таймлайна;
- отображение меток (титры, реклама);
- возможность перемотки:
  - по таймлайну;
  - тапами по экрану.

Проверка проводится в:

- плеере трейлеров;
- плеере основного контента;
- плеере мини-драм;
- плеере загруженного контента.

---

### 3. UI (интерфейс)

- проверка отображения новой иконки в профиле;
- замена "бургер-меню" на иконку "вопрос";
- корректность визуального отображения;
- понятность элемента для пользователя.

---

## Структура тест-кейсов

Каждый тест-кейс включает:

- ID  
- Название  
- Предусловия  
- Шаги выполнения  
- Ожидаемый результат (ОР)  
- Фактический результат (ФР)  
- Окружение  
- Результат  

---

## Окружение

Тестирование проводилось на:

- iOS устройствах  
- "чистой" установке приложения  

---

## Особенности тестирования

- проверка после переустановки приложения;
- тестирование разных типов контента;
- проверка пользовательских сценариев;
- анализ UI/UX изменений;
- проверка стабильности (отсутствие крашей).

---

## Результаты

В ходе тестирования проверено:

- корректное отображение информера;
- стабильность работы плеера;
- корректность UI-изменений;
- работоспособность перемотки и таймлайна.

---

## Примечание

Тест-кейсы направлены на проверку как функциональности, так и пользовательского опыта, включая визуальное восприятие и удобство взаимодействия с приложением.
