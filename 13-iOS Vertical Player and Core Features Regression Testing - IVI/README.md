# 13-iOS Vertical Player and Core Features Regression Testing — IVI

Практический проект по регрессионному тестированию iOS-приложения IVI после обновления ключевых разделов плеера, вертикального контента и пользовательских блоков.

Проект включает проверку стабильности приложения, работы нового функционала и корректности переходов к подписке для пользователей без активной подписки.

## Цель проекта

Проверить корректность работы обновлённого вертикального плеера, режима PiP, галереи серий, раздела настроек, блока «Продолжить просмотр», главной страницы и поведения приложения при изменении сети.

## Объект тестирования

Приложение: IVI  
Платформа: iOS  
Тип тестирования: Manual QA  
Тип проверки: Regression + Functional + UX

---

## Основные направления тестирования

### 1. Вертикальный плеер и PiP

- исправление краша при сворачивании приложения;
- корректность отображения кнопки PiP;
- работа режима «Картинка в картинке»;
- автопереход на следующую серию в PiP;
- корректность системного закрытия PiP;
- переход к пэйволу после окончания бесплатной серии.

---

### 2. Галерея серий (новая реализация)

- отображение текущей серии;
- отображение прогресса просмотра;
- переход к серии с ограничением по подписке;
- корректность отображения значка замка;
- открытие пэйвола при попытке просмотра платной серии.

---

### 3. Настройки плеера (новая реализация)

- стабильность окна настроек;
- переключение вкладок;
- смена качества;
- смена аудиодорожки и субтитров;
- изменение скорости воспроизведения;
- сохранение параметров после изменения;
- работа вертикального режима;
- ограничения качества для пользователей без подписки.

---

### 4. Раздел «Продолжить просмотр»

- корректность возобновления просмотра;
- работа контекстного меню;
- удаление контента;
- отображение прогресса;
- корректность порядка блоков;
- скрытие пустых блоков;
- переход к подписке при попытке просмотра платного контента.

---

### 5. Главная страница

- корректность отображения блоков;
- запуск видео;
- отсутствие визуальных и функциональных дефектов.

---

### 6. ТВ-каналы и работа с сетью

- проверка отсутствия краша при смене Wi-Fi на LTE;
- проверка поведения при отключении интернета;
- стабильность карточки ТВ-канала.

---

## Типы тестирования

- функциональное тестирование;
- регрессионное тестирование;
- тестирование подписочных ограничений;
- UX-тестирование;
- тестирование стабильности;
- тестирование сетевых сценариев;
- тестирование crash-fix.

---

## Результат

В ходе тестирования подтверждена стабильность обновлённых разделов, корректность работы режима PiP, настроек плеера и пользовательских блоков. Проверены сценарии перехода к подписке и отсутствие критических крашей при смене сети.
