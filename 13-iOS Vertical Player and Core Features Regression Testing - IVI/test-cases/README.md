## Test Cases

В данном разделе представлены тест-кейсы, разработанные для регрессионного тестирования iOS-приложения IVI после обновления вертикального плеера и связанных модулей.

Тест-кейсы охватывают проверку стабильности, новой реализации функционала и корректности пользовательских сценариев после внесённых изменений в код.

---

## Покрываемые модули

### 1. Вертикальный плеер и режим PiP

Проверки включают:

- отсутствие краша при сворачивании приложения;
- корректность отображения кнопки PiP;
- работа режима «Картинка в картинке»;
- воспроизведение в фоне и после блокировки экрана;
- корректный выход из PiP;
- автопереход на следующую серию;
- переход к пэйволу после окончания бесплатной серии.

---

### 2. Галерея серий

Проверяется:

- открытие галереи;
- отображение прогресса просмотра;
- отображение иконки воспроизведения;
- отображение значка «замок» у платных серий;
- переход к экрану подписки при выборе закрытой серии.

---

### 3. Настройки плеера

Проверки включают:

- наличие вкладок «Качество», «Аудио и Субтитры», «Настройки», «Ещё»;
- корректность отображения вкладок для пользователей с подпиской и без неё;
- изменение качества видео;
- сохранение параметров после изменения;
- ограничения качества для пользователей без подписки;
- работа вертикального режима.

---

## Типы тестирования

В рамках тест-кейсов применялись:

- регрессионное тестирование;
- функциональное тестирование;
- тестирование crash-fix;
- тестирование подписочных ограничений;
- позитивные и негативные сценарии.

---

## Структура тест-кейсов

Каждый тест-кейс включает:

- ID тест-кейса;
- предусловия;
- шаги воспроизведения;
- ожидаемый результат;
- фактический результат;
- статус выполнения (Passed / Failed);
- окружение тестирования.

---

## Окружение

- Устройство: iPhone 14 Pro Max  
- ОС: iOS 18.6.2  
- Приложение: IVI  

---

Тест-кейсы предназначены для подтверждения стабильности обновлённого функционала и выявления регрессионных дефектов в ключевых пользовательских сценариях.
