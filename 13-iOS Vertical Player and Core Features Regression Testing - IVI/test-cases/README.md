## Тест-кейсы

В данном разделе представлены тест-кейсы, разработанные для регрессионного тестирования iOS-приложения IVI после обновления вертикального плеера и ключевых пользовательских разделов.

Тестирование направлено на проверку стабильности приложения, корректности нового функционала и сохранности пользовательских сценариев после обновления программного кода.

---

## Основные направления тестирования

### 1. Вертикальный плеер и режим PiP

- проверка исправления краша при сворачивании приложения;
- отображение кнопки PiP только во время просмотра серии;
- работа режима «Картинка в картинке»;
- воспроизведение в фоне и после блокировки экрана;
- корректность возврата из PiP;
- автопереход на следующую серию;
- корректное системное закрытие PiP;
- переход к пэйволу после окончания бесплатной серии.

---

### 2. Галерея серий (новая реализация)

- отображение текущей серии;
- отображение иконки воспроизведения и прогресса;
- корректность отображения замка у платных серий;
- переход к экрану подписки при выборе закрытой серии.

---

### 3. Настройки плеера

- стабильность окна настроек;
- переключение вкладок;
- изменение качества видео;
- смена аудиодорожки и субтитров;
- изменение скорости воспроизведения;
- сохранение параметров после смены качества;
- работа вертикального режима;
- ограничения качества для пользователей без подписки.

---

### 4. Раздел «Продолжить просмотр»

- корректность возобновления просмотра;
- работа контекстного меню;
- удаление контента из блока;
- корректность отображения прогресса;
- порядок отображения блоков;
- скрытие пустых разделов;
- переход к подписке при попытке просмотра платного контента.

---

### 5. Главная страница

- корректность отображения блоков;
- запуск видео;
- отсутствие визуальных и функциональных дефектов.

---

### 6. ТВ-каналы и сетевые сценарии

- проверка отсутствия краша при смене Wi-Fi на LTE;
- поведение приложения при отключении сети;
- стабильность карточки ТВ-канала.

---

## Типы тестирования

В тест-кейсах используются следующие виды проверок:

- регрессионное тестирование;
- функциональное тестирование;
- тестирование crash-fix;
- UX-тестирование;
- тестирование подписочных ограничений;
- тестирование сетевых сценариев;
- позитивные и негативные сценарии.

---

## Структура тест-кейсов

Каждый тест-кейс включает:

- идентификатор;
- предусловия;
- шаги воспроизведения;
- ожидаемый результат;
- фактический результат;
- статус выполнения;
- привязку к задаче (например, IOS-XXXX).

Тест-кейсы предназначены для проверки стабильности приложения после обновлений и подтверждения корректности работы ключевых пользовательских сценариев.
