# 16 - IVI iOS Lock Screen Playback & Feature Validation Testing

Практический проект по ручному тестированию мобильного приложения **IVI** для платформы **iOS**.  
Проект посвящён проверке исправленных дефектов, стабильности приложения и тестированию нового функционала воспроизведения контента на заблокированном экране устройства.

Тестирование включало проверку работы различных типов плееров, пользовательских сценариев взаимодействия с контентом и корректности работы обновлённых разделов приложения.

---

## Цель проекта

Подтвердить корректность работы нового функционала и исправленных ошибок, а также убедиться в отсутствии регрессии в ключевых пользовательских сценариях приложения.

---

## Объект тестирования

Мобильное приложение **IVI**

Платформа: **iOS**  
Версия приложения: **26.3.3**  
Устройство: **iPhone 14 Pro Max**  
Версия iOS: **18.6.2**

Тип тестирования: **Manual QA**

---

## Проверяемые функциональные области

В рамках проекта были протестированы следующие функции приложения:

### Исправление краша при вводе текста

Проверено исправление ошибки, при которой приложение падало при вводе текста свайпом в поле авторизации или регистрации.

Проверяемый сценарий:
- открытие формы регистрации;
- ввод текста свайпом.

Ожидаемый результат — приложение не падает, текст корректно вводится.

---

### Работа поля поиска

Проверена корректность работы кнопки очистки поля поиска.

Проверяемый сценарий:
- ввод текста в поисковую строку;
- нажатие на кнопку очистки.

Ожидаемый результат — поле поиска очищается.

---

### Раздел «Буду смотреть»

Раздел был переведён на новый внутренний сервис.

Проверены сценарии:

- добавление контента в раздел;
- корректное отображение статуса добавления;
- удаление контента из раздела;
- отображение списка контента.

---

### Воспроизведение на заблокированном экране

Проверен новый функционал воспроизведения контента при заблокированном экране устройства.

Тестирование проводилось для различных типов плееров:

- ТВ-плеер  
- стандартный видеоплеер  
- плеер мини-драм  
- режим **«Картинка в картинке» (PiP)**

Также проверялись сценарии для пользователей:

- с активной подпиской;
- без активной подписки.

## Артефакты проекта

В репозитории представлены следующие материалы:

- **test-cases** — тест-кейсы с описанием сценариев тестирования  
- **bug-reports** — баг-репорты с обнаруженными дефектами  
- **screenshots** — скриншоты, подтверждающие найденные ошибки  
- **video** — видеозаписи прохождения пользовательских сценариев

---

## Результаты тестирования

В ходе тестирования были выявлены дефекты, связанные с работой нового функционала воспроизведения контента на заблокированном экране.

Основные проблемы:

- отсутствие воспроизведения ТВ-каналов при блокировке экрана;
- отсутствие виджета управления воспроизведением;
- некорректная работа виджета управления в режиме **Picture-in-Picture**.
