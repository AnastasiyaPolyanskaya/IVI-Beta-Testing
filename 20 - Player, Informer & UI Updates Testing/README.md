# 20 - Player, Informer & UI Updates Testing

Практический проект по ручному тестированию мобильного приложения.

В рамках проекта проведено тестирование обновлений, связанных с информером аудиосериалов, компонентами плеера и изменениями пользовательского интерфейса.

---

## Цель проекта

Проверить корректность работы новых и обновленных компонентов приложения, а также убедиться в отсутствии критических ошибок после внесённых изменений.

---

## Объект тестирования

Мобильное приложение

- Платформа: iOS  
- Тип тестирования: Manual QA  

---

## Области тестирования

### 1. Информер аудиосериалов

Проверено:
- отображение информера на "чистой" установке приложения;
- корректность поведения информера при навигации внутри приложения;
- отсутствие визуальных дефектов;
- стабильность (отсутствие крашей).

Ожидаемый результат:
- информер отображается корректно;
- не смещается ("не улетает");
- приложение работает стабильно.

---

### 2. Обновленные компоненты плеера

Проверено:
- работа сикбара (полосы прокрутки);
- отображение таймлайна;
- отображение меток (титры, реклама);
- возможность перемотки:
  - по таймлайну;
  - тапами по экрану.

Проверка проводилась в:
- плеере трейлеров;
- плеере основного контента;
- плеере мини-драм;
- плеере загруженного контента.

Ожидаемый результат:
- все элементы отображаются корректно;
- перемотка работает без ошибок;
- взаимодействие с плеером стабильно.

---

### 3. Изменения UI (иконка профиля)

Проверено:
- замена иконки "бургер" на иконку "вопрос";
- корректность отображения новой иконки в профиле;
- визуальная читаемость элемента.

Ожидаемый результат:
- отображается иконка со знаком вопроса;
- элемент корректно воспринимается пользователем.

---

## Особенности тестирования

Тестирование включало:
- проверку на "чистой" установке приложения;
- проверку пользовательских сценариев;
- анализ UI/UX изменений;
- проверку стабильности приложения.

---

## Результаты тестирования

В ходе тестирования проверено:
- корректность работы информера;
- стабильность обновленного плеера;
- корректность UI-изменений.

Выявленные дефекты (если были):
- фиксируются в bug-reports.

---

## Структура проекта

- test-cases — тест-кейсы  
- bug-reports — баг-репорты  
- screenshots — скриншоты  

---

## Вывод

Обновления затрагивают ключевые пользовательские сценарии (плеер, навигация, информеры), поэтому требуют тщательной проверки.
