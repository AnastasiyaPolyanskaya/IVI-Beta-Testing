# Тест-кейсы - Проект №11  
IVI: Функциональное тестирование iOS (Критические пользовательские сценарии)

В данном разделе размещены тест-кейсы, использованные для функционального тестирования iOS-приложения IVI в рамках проверки исправления критических багов и стабильности ключевых сценариев.

## Описание

Тест-кейсы основаны на требованиях из баг-трекера (задачи с префиксом IOS-) и направлены на валидацию исправления ранее обнаруженных дефектов, а также на проверку стабильности приложения в основных пользовательских сценариях.

Проверяется корректность:
- навигации и UI-элементов;
- работы с аккаунтами (детскими и взрослыми);
- обработки deeplink-ов и внешних ссылок;
- A/B-тестов и режима разработчика;
- рекламных интеграций;
- критических переходов между экранами.

## Покрываемый функционал

### Навигация и UI
- Блок "Сезоны и серии" (выделение активной серии)
- Лендинг подписок (верстка и отступы)
- Раздел "Все подписки"
- Новогодняя айдентика (отсутствие после обновления)

### Работа с аккаунтами
- Детский аккаунт (продление подписки)
- Управление детскими аккаунтами (удаление)
- Переключение между аккаунтами

### Интеграции и стабильность
- Чат поддержки (стабильность при открытии)
- Обработка deeplink-ов (корректная ошибка)
- Интерактивная реклама в плеере (кликабельность)
- A/B-тесты в режиме разработчика

### Критические переходы
- Открытие чата поддержки без крашей
- Удаление аккаунта без блокировки UI
- Переход по deeplink в несуществующий раздел
- Активация/деактивация A/B-тестов

## Типы тестирования
- Функциональное
- Регрессионное
- UI/UX тестирование
- Интеграционное тестирование
- Smoke-тестирование критических путей

## Формат тест-кейсов

Тест-кейсы включают:
- ID задачи из трекера (IOS-XXXXX);
- предусловия и подготовку окружения;
- пошаговые сценарии с четкими действиями;
- ожидаемые результаты согласно требованиям;
- фактические результаты (при отклонениях);
- статус выполнения (Passed/Failed/Blocked).

## Особенности тестирования

1. **Тестирование с разными типами аккаунтов:**
   - Взрослый с подпиской/без подписки
   - Детский аккаунт
   - Неавторизованный пользователь

2. **Подготовка окружения:**
   - Активация конкретных A/B-тестов
   - Создание временных аккаунтов
   - Подготовка тестовых данных (deeplink-ов)

3. **Проверка стабильности:**
   - Отсутствие крашей при критических действиях
   - Корректная обработка ошибок
   - Отзывчивость UI при длительных операциях

## Примечание

Тест-кейсы предназначены для проверки исправления критических багов в приложении IVI на iOS-устройствах и могут быть использованы как:
1. Набор для регрессионного тестирования после исправлений
2. Чек-лист для smoke-тестирования основных сценариев
3. База для создания автоматизированных тестов критических путей

Все тест-кейсы привязаны к конкретным задачам в трекере, что позволяет легко отслеживать историю проверок и статус исправлений.
