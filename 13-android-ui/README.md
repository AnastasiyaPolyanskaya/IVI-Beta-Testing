# Проект №13 — IVI  
## Тестирование обновлённых Android-интерфейсов и сценариев взаимодействия

## Описание

В рамках проекта выполнено комплексное тестирование обновлённых интерфейсных компонентов и ключевых пользовательских сценариев в Android-приложении IVI.

Цель тестирования — проверить корректность отображения пользовательского интерфейса, стабильность взаимодействия с элементами управления, а также соответствие поведения приложения ожидаемой логике при использовании A/B-тестов.

В ходе тестирования проверены:
- новый профиль неавторизованного пользователя;
- режим «Картинка в картинке» (Picture-in-Picture);
- отображение и поведение информеров в плеере;
- сценарий перехода от бесплатного к платному контенту;
- корректность авторизации в процессе просмотра контента.

## Окружение

- **Платформа:** Android  
- **Приложение:** IVI (Android, beta-версия)  
- **Тип пользователя:** неавторизованный / авторизованный пользователь с подпиской  
- **Сетевые условия:** онлайн  
- **A/B-тесты:**
  - `ab_new_profile_android 2`
  - `ab_endscreen_paywall_android 2`

## Проверенные сценарии

### 1. Новый профиль неавторизованного пользователя

**Предусловие:**  
Активирован A/B-тест `ab_new_profile_android 2`.

**Шаги:**
1. Перейти в раздел «Профиль» (кнопка «Войти» в навигационной панели).

**Ожидаемый результат (ANDROID-19199):**
- Отображается обновлённый экран профиля неавторизованного пользователя.
- Визуальное оформление соответствует утверждённому дизайну.
- Элементы интерфейса работают корректно:
  - кнопки активны;
  - переходы выполняются без ошибок;
  - скролл списка подписок работает стабильно.

### 2. Информер режима «Картинка в картинке»

**Описание:**  
Для мини-драм добавлен информер о доступности режима Picture-in-Picture.

**Особенности отображения:**
- информер отображается только в плеере серий (не трейлеров);
- показывается единоразово на одном устройстве;
- может отображаться не с первого входа в плеер из-за наличия других информеров.

**Шаги:**
1. Открыть серию мини-драмы.
2. При необходимости несколько раз выйти из плеера и вернуться обратно.

**Ожидаемый результат (ANDROID-26140):**
- Информер отображается корректно.
- По нажатию на кнопку закрытия информер скрывается.
- По нажатию на кнопку «Попробовать» включается режим Picture-in-Picture.
- При отсутствии взаимодействия информер скрывается автоматически через некоторое время.

### 3. Режим «Картинка в картинке» (Picture-in-Picture)

**Шаги:**
1. Перейти в режим Picture-in-Picture при просмотре мини-драмы.
2. Проверить работу мини-плеера и элементов управления.

**Ожидаемый результат (ANDROID-25337):**
- Воспроизведение контента продолжается без ошибок.
- Мини-плеер отображается корректно.
- Контроллы реагируют на нажатия и выполняют соответствующие действия.

### 4. Экран между доступной и недоступной серией

**Предусловие:**
- Пользователь не авторизован или не имеет активной подписки.
- Активирован A/B-тест `ab_endscreen_paywall_android 2`.

**Шаги:**
1. Открыть сериал с одной бесплатной серией  
   (например, 1 серия 1 сезона сериала «Лейла»).
2. Перемотать серию до титров.

**Ожидаемый результат (ANDROID-25394.1):**
- Отображается экран (или боттомшит) «Следующие серии только по подписке».
- На планшетах и в горизонтальной ориентации экран отображается по центру.

### 5. Авторизация и доступ к платному контенту

**Шаги:**
1. Нажать кнопку покупки подписки.
2. Авторизоваться под пользователем с активной подпиской.
3. Завершить процесс авторизации.

**Ожидаемый результат (ANDROID-25394.2):**
- Пользователь успешно авторизован.
- Следующие серии контента становятся доступными для просмотра.
