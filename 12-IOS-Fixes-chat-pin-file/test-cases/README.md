## Тест‑кейсы — Проект №12 (Обновлённые интерфейсные компоненты IVI)

В данном разделе размещены тест‑кейсы, выполненные в рамках проверки обновлённых интерфейсных компонентов и ключевых сценариев взаимодействия в приложении IVI.

## Основные направления тестирования

Тест‑кейсы охватывают следующие области:

- **Поля ввода текста** — проверка корректности работы и отклика на пользовательские действия в различных формах (поиск, профиль, регистрация, авторизация, шеринг, активация промокодов).
- **Механизм ПИН‑кода** — тестирование сценариев установки и подтверждения ПИН‑кода, обработка ошибок.
- **Локализация метаданных** — проверка отображения размера файла в соответствии с системным языком устройства.
- **Доступность функций в офлайн** — контроль видимости и работоспособности кнопки выхода из аккаунта при отсутствии интернет‑соединения.
- **Чат поддержки** — проверка открытия чата через раздел «Сообщить о проблеме», включая сценарий авторизации внутри чата.

## Формат тест‑кейсов

Каждый тест‑кейс содержит обязательные поля:

- **ID** — уникальный идентификатор тест‑кейса;
- **Название** — краткое описание проверяемой функции/сценария;
- **Предусловия** — условия, необходимые для выполнения тест‑кейса (состояние приложения, настройки устройства, авторизация и т. п.);
- **Шаги воспроизведения** — последовательность действий для проверки сценария;
- **Ожидаемый результат** — описание корректного поведения системы после выполнения шагов.

## Применимость

Тест‑кейсы предназначены для:

- проверки корректности работы обновлённых интерфейсных компонентов;
- верификации исправлений по задачам IOS‑18443, IOS‑18544, IOS‑18551, IOS‑17918, IOS‑18480;
- тестирования кросс‑платформенной совместимости (iOS);
- контроля локализации и адаптивности интерфейса;
- проверки устойчивости работы в различных сетевых условиях (онлайн/офлайн).

Тестирование проводится на реальных устройствах и эмуляторах с актуальными версиями приложения.
