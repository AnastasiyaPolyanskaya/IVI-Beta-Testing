## Test Cases

В данном разделе представлены тест-кейсы, разработанные для проверки исправленных дефектов в мобильном приложении IVI (Android).

Тестирование направлено на подтверждение корректности багфиксов и отсутствие регрессии в смежных пользовательских сценариях.

---

## Покрываемые сценарии

### 1. Блок «Продолжить просмотр»

Проверяется:

- наличие контента в блоке «Продолжить просмотр»;
- корректность вызова контекстного меню;
- удаление контента из блока;
- обновление интерфейса после удаления;
- отсутствие повторного появления удалённого контента.

**Связанная задача:** ANDROID-26253

---

### 2. Бесшовная авторизация (Web → App)

Проверяется:

- авторизация пользователя в мобильной веб-версии ivi.ru;
- переход из веба в приложение через кнопку «Открыть в приложении»;
- корректная передача авторизационных данных;
- автоматическая авторизация пользователя в приложении;
- отсутствие запроса повторного ввода логина и пароля.

**Связанная задача:** ANDROID-26569

---

## Типы тестирования

В рамках тест-кейсов использованы:

- Bug Fix Verification;
- регрессионное тестирование;
- функциональное тестирование;
- кросс-платформенное тестирование (Web → App).

---

## Структура тест-кейсов

Каждый тест-кейс включает:

- ID тест-кейса;
- предусловия;
- шаги воспроизведения;
- ожидаемый результат;
- фактический результат;
- статус выполнения;
- окружение тестирования.
