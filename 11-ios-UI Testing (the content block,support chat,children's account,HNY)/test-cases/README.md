# Тест-кейсы - Проект №11  
IVI: Функциональное тестирование iOS (Критические пользовательские сценарии)

В данном разделе размещены тест-кейсы, использованные для функционального тестирования iOS-приложения IVI версии 26.1.2 в рамках проверки исправления критических багов и стабильности ключевых сценариев.

## Описание

Тест-кейсы основаны на требованиях из баг-трекера (задачи с префиксом IOS-) и направлены на валидацию исправления ранее обнаруженных дефектов, а также на проверку стабильности приложения в основных пользовательских сценариях.

Проверяется корректность:
- навигации и UI-элементов;
- работы с аккаунтами (детскими и взрослыми);
- отображения новогодней айдентики после отката обновления;
- режима разработчика и A/B-тестов;
- критических переходов между экранами.

## Покрываемый функционал

### Навигация и UI
- Блок "Сезоны и серии" (выделение активной серии)
- Лендинг подписок Иви/Изи Иви (верстка и отступы)
- Новогодняя айдентика (отсутствие после отката обновления)
- Раздел уведомлений (отображение контента)

### Работа с аккаунтами
- Управление детскими аккаунтами (удаление и навигация)
- Раздел "Мои аккаунты" -> "Детские"
- Возврат на корректный экран после операций с аккаунтами

### Стабильность и режим разработчика
- Чат поддержки (стабильность при открытии)
- A/B-тесты в режиме разработчика (отсутствие специфичных тестов)
- Видимость всех элементов в списках A/B-тестов

### Критические переходы
- Открытие чата поддержки без крашей и зависаний
- Удаление детского аккаунта без блокировки UI
- Навигация в разделе "Поделиться подпиской"
- Навигация в разделе уведомлений для неавторизованного пользователя

## Типы тестирования
- Функциональное тестирование
- Регрессионное тестирование
- UI/UX тестирование
- Smoke-тестирование критических путей

## Формат тест-кейсов

Тест-кейсы включают:
- ID тест-кейса (ивпркд11.X);
- предусловия и подготовку окружения;
- пошаговые сценарии с четкими действиями;
- ожидаемые результаты согласно требованиям;
- фактические результаты (при отклонениях);
- окружение тестирования;
- статус выполнения (Passed/Failed).

## Особенности тестирования

1. **Тестирование с разными типами аккаунтов:**
   - Взрослый с активной подпиской
   - Детский аккаунт (временный для удаления)
   - Неавторизованный пользователь

2. **Окружение тестирования:**
   - Устройство: iPhone 14 Pro Max
   - ОС: iOS 18.6.2
   - Версия приложения: 26.1.2

3. **Проверка стабильности:**
   - Отсутствие крашей при критических действиях
   - Корректная навигация между экранами
   - Отзывчивость UI при операциях с аккаунтами

## Структура тест-кейсов

### Раздел 1: Навигация и UI (2 тест-кейса)
- Проверка выделения активной серии
- Проверка отступов на лендинге подписки

### Раздел 2: Стабильность приложения (1 тест-кейс)
- Проверка работы чата поддержки

### Раздел 3: Работа с аккаунтами (1 тест-кейс)
- Проверка удаления детского аккаунта

### Раздел 4: Контент и визуальное оформление (2 тест-кейса)
- Проверка отсутствия новогодней айдентики в двух разделах

### Раздел 5: Режим разработчика (2 тест-кейса)
- Проверка отсутствия специфичных A/B-тестов
- Проверка видимости элементов в A/B-тестах
