# bug-reports

В данной папке собраны отчеты о дефектах (bug reports), обнаруженных в ходе тестирования проекта №11.

## Содержание

### Структура отчетов
Каждый баг-репорт содержит:
- **ID дефекта** (присвоенный в трекере)
- **Заголовок** (краткое описание проблемы)
- **Шаги воспроизведения** (последовательные действия)
- **Фактический результат** (что происходит на самом деле)
- **Ожидаемый результат** (что должно происходить по требованиям)
- **Окружение** (устройство, ОС, версия приложения)
- **Серьезность** (Critical/Major/Minor/Cosmetic)
- **Приоритет** (High/Medium/Low)
- **Прикрепленные файлы** (скриншоты, видео, логи)

### Категории дефектов
1. **UI/UX проблемы** (некорректное отображение, верстка)
2. **Функциональные дефекты** (нарушение логики работы)
3. **Проблемы стабильности** (краши, зависания, утечки памяти)
4. **Интеграционные ошибки** (работа с внешними сервисами)
5. **Контентные проблемы** (некорректный контент, тексты)
