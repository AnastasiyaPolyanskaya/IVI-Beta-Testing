# Баг-репорты - Проект №10 (Удаление контента)

В данном разделе размещены баг-репорты, выявленные в ходе тестирования функционала удаления загруженного контента в Android-приложении IVI.

## Содержимое

- **Баг-репорт_ИВИ10.1.docx**

В баг-репорте зафиксирован следующий дефект:

- Появление ассерт-сообщения при удалении загруженного контента после перезапуска приложения;
- Отсутствие ожидаемого попапа подтверждения удаления;
- Нарушение пользовательского сценария работы с загрузками.

## Описание дефекта

**ID бага:** инпркд10.1  
**Название:** Появление ассерта при удалении загруженного контента  
**Краткое описание:** При выполнении сценария удаления ранее загруженного контента после перезапуска приложения вместо стандартного UI-поведения появляется ассерт-сообщение об ошибке с техническими кнопками.

## Детали

- **Ожидаемое поведение:** Пользователь может удалить контент через стандартный интерфейс без технических сообщений
- **Фактическое поведение:** Появляется ассерт с кнопками "Продолжить без ассертов" и "Сохранить лог"
- **Влияние на пользователя:** Нарушен стандартный сценарий удаления контента, пользователь сталкивается с технической ошибкой

## Применимость

Баг-репорт относится к мобильной версии IVI для Android (версия 26.1.1) и описывает критический дефект, влияющий на основной функционал приложения — управление загруженным контентом.

- Версия приложения: 26.1.1
