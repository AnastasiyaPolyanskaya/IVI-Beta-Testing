# 15-UI Terminology Update & GUP Functional Testing - IVI

Практический проект по проверке текстовых изменений в интерфейсе и корректности работы блока ГУП (главного управляющего профиля) после внесения изменений на уровне программного кода.

Проект направлен на верификацию отображения обновлённой терминологии («Сертификат» → «Промокод») и проверку логики вывода информации о подписке.

---

## Цель проекта

Подтвердить корректность:

- замены термина «Сертификат» на «Промокод» во всех указанных разделах продукта;
- отображения текстов и сообщений об ошибках;
- работы ГУП при различных состояниях подписки;
- функционала возобновления продления подписки.

---

## Объект тестирования

- Приложение: IVI  
- Тип тестирования: Manual QA  
- Формат: UI Verification / Functional Testing / Regression  

---

## Проверяемые изменения

### 1. Обновление терминологии («Сертификат» → «Промокод»)

Проверено корректное отображение текста в следующих разделах:

- Профиль (кнопка «Активировать промокод»);
- Раздел активации промокода (все элементы экрана);
- Экран успешной активации («Промокод активирован»);
- Ошибка при введении невалидного промокода («Промокод не активирован»);
- Ошибка при активации сертификата с некорректной картой («При активации промокода что-то пошло не так»).

Ожидаемый результат (VISION-18562):  
Во всех указанных местах отображается термин «Промокод».

---

### 2. Изменения логики отображения ГУП

Проверено корректное отображение информации в ГУП при различных состояниях подписки:

- активная подписка;
- подписка в добиливании;
- просроченная подписка;
- отсутствие подписки.

Также проверена работа функции возобновления продления подписки.

Ожидаемый результат (VISION-18524):  
Информация отображается корректно, без ошибок и некорректных данных, функционал работает стабильно.

---

## Результат

Проект демонстрирует навыки:

- проверки UI-изменений;
- верификации текстовых обновлений;
- регрессионного тестирования;
- проверки логики отображения подписки;
- анализа пользовательских состояний.
